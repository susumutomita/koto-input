# かな形態巡回（Tab 連打でひらがな・カタカナ切替） 仕様書

## 概要

Tab キーの連打で composition のかな形態を巡回する。1 回目の Tab はローマ字をひらがなへ（既存）、2 回目はカタカナへ、3 回目はひらがなへ戻る。AI 不要・決定論・即時。カタカナ語（外来語・オノマトペ等）を確実に打てるようにする。

## 背景（実機フィードバック）

カタカナに変換したい場合の手段が無い。Shift + Space の連打は「同じ形態での別候補の再抽選」（ADR-0008）に割り当て済みのため、形態切替は Tab 連打に割り当てる（ヒアリングで決定。再抽選・候補切替 ↑/↓ とは衝突しない）。

## 振る舞い

1. composing 中の Tab（1 回目）: 従来どおり `RomajiKanaConverter` でひらがな化（保護語は除外）。
2. 直後の Tab（2 回目）: 表示テキストのひらがなをカタカナへ機械変換する。ひらがな以外の文字（ASCII・保護語・記号・長音符）は変更しない。
3. さらに Tab: ひらがなへ戻る。以降カタカナ ⇄ ひらがなを巡回する。
4. 巡回状態は他の編集（挿入・削除・カーソル移動を伴う編集）・AI 変換・commit・cancel・Escape でリセットされる。リセット後の Tab は再び「ひらがな化」から始まる。
5. 変換（converting / converted）中の Tab は従来どおり編集として扱う（変換キャンセル等の既存規則を変えない）。
6. composition が無いときの Tab は従来どおり消費しない（ターミナルの補完を奪わない）。

## 受け入れ基準

- [ ] `onnna` → Tab → 「おんな」 → Tab → 「オンナ」 → Tab → 「おんな」（テスト）。
- [ ] 保護語・ASCII・記号はカタカナ化で変化しない（`Claude Code wo testo` → Tab ×2 → 「Claude Code ヲ テスト」相当。保護語原文維持）（テスト）。
- [ ] ゔ・小書きかな・ゕゖ を含む全ひらがな範囲が正しく往復する（テスト）。
- [ ] 編集を挟むと巡回状態がリセットされ、次の Tab はひらがな化になる（テスト）。
- [ ] 既存の Tab（1 回目のひらがな化）・Shift + Space 再抽選・↑/↓ 候補切替の挙動が回帰しない（既存テスト green）。
- [ ] CI green。実機で `katakananihenkan` → Tab ×2 → 「カタカナニヘンカン」を確認。

## 技術設計

- `RomajiKanaConverter` にひらがな ⇄ カタカナの決定論マッピングを追加する（U+3041–U+3096 ⇄ U+30A1–U+30F6 のコードポイント変換。長音符・句読点・濁点記号は不変）。
- `CompositionState` に巡回状態（例: `kanaCycleForm: KanaForm?`、nil = 非巡回）を追加し、`normalizeToKana` が設定・巡回し、他の編集・変換・リセット経路でクリアする。
- キールーティング（InputController の Tab）は変更しない。reducer 内の `normalizeToKana` の解釈だけが変わる。

## スコープ外

- 全角カタカナ ⇄ 半角カタカナ。
- AI 変換候補への形態バリエーション自動追加（ヒアリングで見送り）。
- Shift + Space の意味変更（再抽選のまま）。
