# 多言語変換キー 仕様書

## 概要

Ctrl + Shift + 言語キーの同時押しで、composition のローマ字・かな・日本語をターゲット言語（英語・中国語・韓国語・フランス語・ドイツ語・スペイン語）の自然な文へ AI 変換する。従来の IME には原理的に存在しない「打った場所にそのまま訳文が入る」体験を、オンデバイスモデルだけで実現する。

## ユーザーストーリー

- ターミナルで Claude Code / Codex を使う開発者として、日本語で考えた指示やコミットメッセージを英語で書くために、ローマ字のまま打って Ctrl + Shift + E で英語に変換したい。
- 同じ開発者として、文の一部だけ英語にしたい（コミットメッセージの summary だけ英語等）ので、composition 単位で変換先言語を選びたい。
- 海外メンバーとチャットする開発者として、同じ操作で中国語・韓国語・欧州語にも変換したい。

## ヒアリング決定事項

1. **発動方法**: 修飾キー同時押し（Ctrl + Shift + 言語キー）。変換後の追い打ち単独キーは次の文字入力と衝突するため不採用。
2. **言語キー割当（初期リリースで 6 言語フルセット）**: E = 英語、C = 中国語（簡体字）、K = 韓国語、F = フランス語、G = ドイツ語、S = スペイン語。いずれもオンデバイスモデルの対応言語。
3. **文体**: 自然な訳（意味を保ちつつターゲット言語として自然な文。Claude Code への指示にそのまま使える品質を狙う）。

## 振る舞い

1. **composing 中** に Ctrl + Shift + E: 現在の composition を原文スナップショットとして英語へ変換する（Shift + Space と同じライフサイクル。表示はローマ字のまま → 変換完了で訳文に置換）。
2. **converted 状態**で:
   - 同じ言語キー連打: 同じ原文から attempt + 1 で再抽選（既存の再変換機構と同じ。attempt 0 は greedy、1 以降は温度 0.8）。
   - 別の言語キー: その言語へ attempt 0 から変換し直す（原文スナップショットは維持）。
   - Shift + Space: 日本語変換へ戻す（attempt 0 から）。
3. **Escape**: 従来どおり元のローマ字テキストへ復元する。
4. **Tab**（即時かな化）・編集・commit・type-ahead は従来どおり言語と無関係に動く。
5. composition が無いときの Ctrl + Shift + 言語キーは消費せずアプリへ通す（ターミナルのショートカットを奪わない）。
6. かな正規化の前段は翻訳でも適用する（モデルへは `modelInputText` を渡す。分かち書きなしローマ字の弱点は言語によらないため）。
7. 保護語・頭字語（長さ 2 以上の大文字連続）は翻訳でも原文どおり維持する（識別子・コマンド名は訳さない）。

## 非機能要件

- パフォーマンス: 言語別の LanguageModelSession は初回要求時に生成しキャッシュする。日本語セッションの prewarm（composition 開始時）は現状維持。翻訳セッションの prewarm は行わない（6 言語分の常駐を避ける）。
- セキュリティ・プライバシー: 完全オンデバイス。入力テキストをログ・デバイス外へ出さない方針を維持する（ADR-0002）。
- アクセシビリティ: 表示挙動（marked text の下線のみ）は現状から変更しない。

## 技術設計

- **ConversionTarget**: `enum ConversionTarget: String, CaseIterable, Sendable`（`japanese` / `english` / `chineseSimplified` / `korean` / `french` / `german` / `spanish`）を KotoCore に追加する。
- **CompositionCommand**: `.requestConversion` を `.requestConversion(ConversionTarget)` に拡張する（既存呼び出しは `.japanese`）。
- **CompositionTransition / CompositionState**: 状態に現在の変換ターゲットを保持し、converted からの再要求が「同 target なら attempt + 1、異 target なら attempt 0」になるよう reduce を拡張する。原文スナップショット・prefix 検査・stale 拒否（compositionID / requestID / revision）は共通。
- **ConversionRequest**: `target: ConversionTarget` を追加する。`modelInputText`（かな化）は全 target 共通。
- **PromptBuilder**: `instructions(settings:target:)` に分岐を追加する。日本語は現行の instructions。翻訳 target は「[ROLE] translation engine」「自然なターゲット言語へ訳す」「保護語・頭字語・コード・パスは原文維持」「入力に無い引用符・括弧で包まない」「訳文のみ返す」を言語名付きで指示し、few-shot は忠実な訳の例を 1 つ入れる（言い換え教示を避ける Issue 22 の教訓を踏襲）。
- **AppleFoundationModelsProvider**: instructions が target ごとに異なるため、target をキーにしたセッションキャッシュへ拡張する。greedy / 温度 0.8 の attempt 規則は共通。
- **ConversionOutputValidator**: 共通検証（改行 trim・空・膨張率・保護語・頭字語）は全 target に適用する。日本語固有の検証（末尾句点 strip・鉤括弧 unwrap）は `.japanese` のみに適用する。
- **InputController**: `essentialFlags == [.control, .shift]` かつ `charactersIgnoringModifiers` が言語キーに一致したら `.requestConversion(target)` を発行する。キーコードではなく文字で判定し、レイアウト差を吸収する。
- **設定**: 言語キーマップは初期リリースでは固定。`ConversionSettings` への持ち込み（カスタマイズ）は将来拡張。
- データモデル / UI コンポーネント: 追加なし（marked text の表示は既存のまま）。

## スコープ外

- オンデバイスモデル非対応言語（アラビア語等）。Translation framework との 2 段変換は別機能として検討する。
- 言語キーマップのカスタマイズ UI・設定化。
- composition 内の部分選択だけを訳す操作（変換は composition 全体が対象。一部だけ訳したい場合は分けて確定する運用）。
- 訳調（フォーマル度等）の設定。
- 翻訳セッションの prewarm 最適化。

## 備考

- 変換先が日本語以外でも、Escape 復元・type-ahead・stale 拒否のセマンティクスは既存の状態機械をそのまま使う。本機能は「プロンプトと検証の言語分岐 + キールーティング」であり、状態機械の構造変更を伴わない。
